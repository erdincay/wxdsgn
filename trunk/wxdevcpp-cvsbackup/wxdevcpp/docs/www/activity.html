2004-11-10

00.30am

Lot of people are taking initiative to report bugs. Thank you all for submitting those bug reports.

I fixed couple of big show stoppers. The following are the bugs that are fixed today :
1)There was a Code generation bug in the StaticBoxSizer.
2)The Code generator for the wxHTMLWindow is adding a problematic code.
3)Invisible Components are added as normal control. With the current fix, the Invisible components act as Delphi Like invisible components.
4)Code Generation of check Menu Item and Radio Menu Item had some problem.
5)Changed the Template Loading function to use the Custom properties (Like Debug, Profile and Heuristic Compilation Options turned On by default)-This will fix the issue with program crashing intermittently when adding or removing controls.

After getting enough reports about the 6.4 release, I'll release 6.5 beta by the end of this week.

2004-11-09

Yesterday I got bunch of bug reports from couple of people (Rick being one among them). Most of the bugs are known issues. Julian Smart was asked me to fix this problem for him so that people can use DialogBlock as the main designer for wx-devcpp. I hope he will happy with this fix. There was also a bug in the wxHTMLWindow code generation. I fixed the issue and it will be in the next beta. I'll wait for couple of days and release a new version.
