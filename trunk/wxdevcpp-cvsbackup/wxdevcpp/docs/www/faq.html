What is wx-devcpp ?

wx-devcpp is an extension of DevC++ with a Form Designer for the Amazing Multiplatform toolkit wx-Widgets. The source is based on Sep 18th 2004 of DevC++.

Where can I get Help on wx-devcpp?

Please post your question in the wxForum. You can use this link (http://www.solidsteel.nl/users/wxwidgets/viewforum.php?f=28) to check the forum.

Why should I donate money to wx-Widgets Foundation instead of wx-devcpp?

More than 10 years of development has gone into wx-Widgets development. wx-devcpp is just a wrapper to explore and show the full potential of wx-Widgets, so I would really like you to send couple of bucks to the true genius who are responsible for making the multiplatform programming a reality.
If you donate money to wx-widgets, you will automatically see a sharp increase in the quality of wx-devcpp.

Which language wx-devcpp is written with?

wx-devcpp is written with Borland Delphi 6.0. Lot of third party components like jvcl, extlib etc are used.

Can I get the wx-devcpp source Code?

Sure, Check wx-devcpp 's CVS. Check http://cvs.sourceforge.net/viewcvs.py/wxdsgn/wxdevcpp/

Will there be a release for Linux?

You have to ask this question to the core Dev-C++ developers. Since wx-devcpp is based on the Dev-C++ source code, a linux will be a possible only if we have a Linux version of Dev-C++.

I'm not able to see the Design Dialog, what is happening ?

There is a problem in the pre-6.4 releases wxForm files. Please open up the text editor and Remove the line

BorderStyle = bsDialog

This should allow the wxForm to be edited properly.

Why does wx-devcpp compiled program crashes after adding a control ?

It seems there is a problem with the makefile dependency. Uncheck the "Use Fast but imperfect dependency generation" option (you can see that at the top of the dialog) in the Compiler Options. (ps : This issue is fixed in beta 6.6)
